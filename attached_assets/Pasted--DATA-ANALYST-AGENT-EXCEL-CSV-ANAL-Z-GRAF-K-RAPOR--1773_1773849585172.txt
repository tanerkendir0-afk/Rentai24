# DATA ANALYST AGENT — EXCEL/CSV ANALİZ + GRAFİK + RAPOR
# ============================================================
# Bu talimatı Replit Agent'a olduğu gibi yapıştır.
# ============================================================

Data Analyst Agent'ı (data-analyst) gerçek veri analizi yapabilecek şekilde güçlendirmek istiyorum.
Şu an sadece iç CRM verisini sorguluyor (query_leads, query_actions, query_campaigns).
Müşterinin kendi dosyalarını (Excel, CSV) yükleyip analiz etme yeteneği YOK.

KOBİ'ler için en değerli özellik: "Excel'imi yükle, analiz et, grafik çıkar, rapor ver."

## YAPILACAKLAR:

### 1. DATABASE — Yüklenen dosya takibi

```sql
CREATE TABLE IF NOT EXISTS uploaded_files (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id),
  original_name VARCHAR(500) NOT NULL,
  stored_path TEXT NOT NULL,
  file_type VARCHAR(20) NOT NULL,  -- 'xlsx', 'xls', 'csv', 'tsv'
  file_size INTEGER,
  row_count INTEGER,
  column_names JSONB,              -- ["Tarih", "Ürün", "Miktar", "Tutar"]
  summary JSONB,                   -- otomatik özet istatistikler
  uploaded_at TIMESTAMP DEFAULT NOW(),
  expires_at TIMESTAMP             -- 24 saat sonra otomatik sil
);
```

Drizzle schema'ya da ekle.

### 2. BACKEND — Dosya Yükleme API

```
POST   /api/files/upload        — Excel/CSV yükle (multipart/form-data)
GET    /api/files                — Yüklenen dosyaları listele
GET    /api/files/:id/preview    — İlk 20 satırı önizle
GET    /api/files/:id/download   — Dosyayı indir
DELETE /api/files/:id            — Dosyayı sil
```

Upload endpoint:
- Max 10MB
- Kabul edilen formatlar: .xlsx, .xls, .csv, .tsv
- Yükleme sonrası otomatik: satır sayısı, kolon adları, temel istatistikler hesapla
- 24 saat sonra otomatik temizlik (cron job veya expiry check)
- Dosyalar `uploads/data-analyst/` dizinine kaydedilsin

### 3. BACKEND — Veri Analiz Servisi

`server/services/dataAnalysisService.ts` oluştur:

```typescript
import * as XLSX from 'xlsx';  // xlsx paketi (SheetJS)
import * as Papa from 'papaparse';  // CSV parser

interface AnalysisResult {
  summary: {
    rowCount: number;
    columnCount: number;
    columns: ColumnInfo[];
  };
  statistics: Record<string, ColumnStats>;
  insights: string[];
}

interface ColumnInfo {
  name: string;
  type: "number" | "text" | "date" | "boolean" | "mixed";
  sampleValues: any[];
  nullCount: number;
  uniqueCount: number;
}

interface ColumnStats {
  min?: number;
  max?: number;
  mean?: number;
  median?: number;
  sum?: number;
  stddev?: number;
  topValues?: { value: string; count: number }[];  // text kolonlar için
}

// Excel/CSV dosyasını parse et
async function parseFile(filePath: string): Promise<any[][]> { ... }

// Temel istatistikler
async function analyzeData(data: any[][]): Promise<AnalysisResult> { ... }

// Kolon bazında gruplama ve toplama
async function groupBy(data: any[][], groupCol: string, aggCol: string, aggFunc: "sum" | "avg" | "count" | "min" | "max"): Promise<any[]> { ... }

// Filtreleme
async function filterData(data: any[][], column: string, operator: string, value: any): Promise<any[][]> { ... }

// Pivot tablo
async function pivotTable(data: any[][], rowField: string, colField: string, valueField: string, aggFunc: string): Promise<any[][]> { ... }

// Korelasyon analizi (iki sayısal kolon arası)
async function correlate(data: any[][], col1: string, col2: string): Promise<{ correlation: number; interpretation: string }> { ... }

// Trend analizi (tarih kolonuna göre)
async function trendAnalysis(data: any[][], dateCol: string, valueCol: string): Promise<{ trend: string; change: number; periods: any[] }> { ... }

// Anomali tespiti (outlier detection)
async function detectAnomalies(data: any[][], column: string): Promise<{ anomalies: any[]; threshold: number }> { ... }
```

Bağımlılık: `npm install xlsx papaparse`

### 8. TOOL HANDLER'LARI

`server/agentTools.ts`'deki executeToolCall fonksiyonuna handler'ları ekle:

```typescript
case "list_uploaded_files":
  return await dataAnalysisService.listFiles(userId);

case "analyze_file":
  return await dataAnalysisService.analyzeFile(userId, args.file_id, args.focus_columns);

case "query_file_data":
  return await dataAnalysisService.queryData(userId, args);

case "create_chart":
  const chartConfig = await dataAnalysisService.createChart(userId, args);
  // chartConfig'i response'a özel formatta ekle (frontend'de render için)
  return { success: true, chart: chartConfig, message: "Grafik oluşturuldu." };

case "compare_columns":
  return await dataAnalysisService.compareColumns(userId, args.file_id, args.column_1, args.column_2);

case "detect_anomalies":
  return await dataAnalysisService.detectAnomalies(userId, args.file_id, args.column, args.sensitivity);

case "trend_analysis":
  return await dataAnalysisService.trendAnalysis(userId, args);

case "generate_analysis_report":
  // PDF veya Excel rapor üret, file_ref döndür
  return await dataAnalysisService.generateReport(userId, args);

case "export_filtered_data":
  return await dataAnalysisService.exportData(userId, args);
```

### 9. SYSTEM PROMPT GÜNCELLEMESİ

Data Analyst agent'ın system prompt'una ekle:

```
## Veri Analiz Yetenekleri

Sen Excel ve CSV dosyalarını analiz edebilen güçlü bir veri analistisin.

### Dosya yüklendiğinde:
1. Önce analyze_file çağır → genel bakış sun
2. İlginç pattern'leri belirt
3. Önerilen analizleri sırala (trend, anomali, gruplama vb.)
4. Grafik öner

### Analiz akışı:
- "Bu Excel'i analiz et" → analyze_file → özet + öneriler
- "Şehir bazında satışlar?" → query_file_data (group_by: Şehir, aggregate: sum)
- "Grafik çıkar" → create_chart (otomatik en uygun tip)
- "Trend nasıl?" → trend_analysis
- "Anomali var mı?" → detect_anomalies
- "Rapor oluştur" → generate_analysis_report (PDF)
- "Filtrelenmiş veriyi Excel'e aktar" → export_filtered_data

### Grafik seçimi:
- Kategorik karşılaştırma → bar chart
- Zaman serisi → line chart  
- Oran/pay dağılımı → pie/doughnut
- İki değişken ilişkisi → scatter
- Kümülatif trend → area chart
- Çok kategorili → horizontal bar

### Türkçe sayı formatı:
- 1.250.000,50 ₺ (binlik nokta, ondalık virgül)
- Grafiklerde TL formatı kullan

### Proaktif analiz:
Sadece sorulan soruyu yanıtlama, ayrıca:
- İlginç pattern'leri belirt ("Mart ayında %40 düşüş dikkat çekici")
- Olası nedenleri tartış
- Aksiyon önerileri sun
- Karşılaştırma yap ("Geçen yılın aynı dönemine göre %15 artış")
```

## ÖNEMLİ NOTLAR

- xlsx paketi (SheetJS) hem .xlsx hem .csv okuyabilir
- Büyük dosyalarda (>5000 satır) streaming parse kullan, tamamını memory'e alma
- Chart config'i JSON olarak agent response'ta döndür, frontend render etsin
- generate_analysis_report tool'u mevcut PDF generator'ı (pdfBrandingService) kullanabilir
- Mevcut tool'lar (query_leads, query_actions vb.) KALSIN — bunlar iç platform verisi için
- Yeni tool'lar dış veri (müşterinin dosyaları) için
