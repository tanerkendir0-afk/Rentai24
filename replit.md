# RentAI 24

## Overview
RentAI 24 is envisioned as the world's first AI staffing agency website, offering businesses a platform to browse, rent, and manage pre-trained AI agents. These agents are designed to fill various roles such as customer support, sales, and bookkeeping. The project aims to provide flexible, on-demand AI talent, reducing the overhead associated with traditional hiring. The platform seeks to tap into the growing demand for AI solutions in business operations, offering a streamlined process for deploying AI workers 24/7.

## System Architecture
The RentAI 24 platform is built with a modern web stack. The frontend utilizes React, TypeScript, and Tailwind CSS, bundled with Vite, emphasizing a dark mode UI with a navy background and blue-to-violet gradient accents. The backend is powered by Express.js, with PostgreSQL (Neon serverless) as the primary database, managed through Drizzle ORM. AI capabilities are integrated via OpenAI GPT-4o, leveraging Replit AI Integrations for core functionalities.

**Key Features:**

*   **AI Worker Catalog:** A dedicated section showcasing 9 distinct AI agents, each with a specific persona (e.g., Ava the Customer Support Agent, Rex the Sales Development Rep, Reno the Real Estate Agent).
*   **User Authentication & Authorization:** Session-based authentication using `express-session` and `bcrypt` for secure user management, including login, registration, and protected routes.
*   **Customer Dashboard:** A personalized dashboard for users to manage rented AI workers, monitor usage statistics, and handle billing.
*   **Admin Panel:** A password-protected interface for administrators to manage RAG documents, fine-tune AI models, export training data, download agent rules, and monitor system activity. Features a **Boss AI** tab — an AI assistant (GPT-4o) that knows the entire platform architecture, can query live database stats (users, rentals, costs, agent usage), and provides development guidance. Boss has 6 tools: `query_platform_stats`, `query_agent_performance`, `query_agent_usage`, `query_recent_activity`, `query_cost_breakdown`, `get_system_health`. Boss conversations are automatically saved to the `boss_conversations` DB table with topic, messages, timestamp, and tool usage tracking. The History panel allows browsing, loading, renaming, and deleting past conversations. APIs: `GET/POST /api/admin/boss-conversations`, `PATCH/DELETE /api/admin/boss-conversations/:id`.
*   **AI Tooling System:** A generalized tool registry allows AI agents to perform specific actions (e.g., sending emails, creating tickets, generating images). This includes tools for sales (Rex), customer support (Ava), scheduling (Cal), data analysis (DataBot), social media (Maya), bookkeeping (Finn), HR (Harper), e-commerce (ShopBot), and real estate (Reno).
*   **RAG & Fine-tuning:** Supports uploading documents for RAG, chunking, embedding using `text-embedding-3-small`, and vector storage via `pgvector`. It also facilitates OpenAI fine-tuning for specialized agent models.
*   **Training Data Export & Agent Rules PDF:** Admin panel includes a "Training Data" tab to generate and download OpenAI fine-tuning compatible JSONL files per agent (with filters: tool usage only). A "Download Agent Rules PDF" button generates a comprehensive PDF document with all 9 agents' rules, tools, forbidden zones, and behavior guidelines. Service: `server/trainingDataService.ts`. Admin APIs: `/api/admin/agent-rules-pdf`, `/api/admin/agents/:agentType/export-training-data`, `/api/admin/agents/:agentType/download-training-data`, `/api/admin/validate-training-data`.
*   **Image Credits & Uploads:** Users can purchase image credits for AI image generation (DALL-E 3) and stock photo searches. The platform supports in-chat image uploads.
*   **Agent Collaboration (Brainstorming):** Admin panel "Collaboration" tab allows all 9 agents to be brought together for team brainstorming. Each agent provides their unique perspective (Ava: support, Rex: sales, Maya: marketing, etc.) on any topic, then Boss AI synthesizes all perspectives into a unified action plan. Agents can be selectively included/excluded. Uses `gpt-4o-mini` for individual perspectives and `gpt-4o` for synthesis. All token costs are tracked under `operation_type='collaboration'`. Sessions are auto-saved to `collaboration_sessions` DB table with topic, synthesis, agent responses, cost, and token data. History panel allows loading and deleting past sessions. APIs: `POST /api/admin/agent-collaboration`, `GET /api/admin/collaboration-sessions`, `DELETE /api/admin/collaboration-sessions/:id`.
*   **Spend Analysis Dashboard:** Admin panel "Spend Analysis" tab provides comprehensive cost analytics: total spend, tokens, avg cost/request, per-agent breakdown with progress bars and percentages, cost by model, cost by operation type (chat/tool_call/collaboration), daily trend chart (last 30 days), and collaboration session costs. API: `GET /api/admin/spend-analysis`.
*   **Token Cost Tracker:** Admin panel tracks all OpenAI API token usage per request — model, prompt/completion tokens, USD cost. Summary and detailed views with $0.01+ expensive request filter. DB table: `token_usage`. Admin APIs: `/api/admin/token-usage/totals`, `/summary`, `/detailed`.
*   **Token Optimization System:** Smart model routing (simple messages → gpt-4o-mini, complex → gpt-4o), conversation history summarization (last 6 messages kept, older summarized via gpt-4o-mini with session-based caching), compressed system prompts (~40-50% shorter), RAG threshold raised to 0.5 with max 3 chunks, conditional tool filtering based on message keywords. Admin "Token Optimization" tab shows avg prompt/total tokens, model distribution, mini usage %, estimated savings, and 7-day trends. API: `GET /api/admin/token-optimization`. Key functions: `routeModel()`, `summarizeConversationHistory()`, `getRelevantToolsForMessage()` in `server/agentTools.ts`.
*   **Server-Side Chat Persistence:** Chat conversations are stored in the PostgreSQL database via the `conversations` table (userId, agentType, visibleId, title) linked to `chat_messages` (sessionId = visibleId). Frontend uses react-query to fetch/create/delete conversations from the API instead of localStorage. Messages persist across browser sessions and devices. APIs: `GET /api/conversations?agentType=X`, `POST /api/conversations`, `PATCH /api/conversations/:id`, `DELETE /api/conversations/:id`, `GET /api/conversations/:visibleId/messages`.
*   **Team Members System:** Users can add, edit, and delete team members (name, email, position, department, skills, responsibilities, phone) in Settings. AI agents are aware of all team members and can reference them when sending emails or coordinating tasks. Team member context is injected into the chat system prompt. DB table: `team_members`. APIs: `GET/POST /api/team-members`, `PATCH/DELETE /api/team-members/:id`.
*   **Per-User Gmail:** Each user can connect their own Gmail via App Password in Settings. AI agents will use the user's Gmail for sending/receiving emails instead of the system default. Credentials stored in `users` table (`gmailAddress`, `gmailAppPassword`). APIs: `GET/POST/DELETE /api/settings/gmail`. Service: `setActiveUserGmail()` in `gmailService.ts`.
*   **Social Media Accounts:** Users can connect Instagram, Twitter/X, LinkedIn, Facebook, TikTok, YouTube accounts in Settings. Maya (Social Media AI) automatically checks connected accounts and suggests connecting more on first interaction. Connected accounts are injected into Maya's context so she can create content tailored to specific profiles. DB table: `social_accounts`. APIs: `GET/POST /api/social-accounts`, `PATCH/DELETE /api/social-accounts/:id`. Maya tool: `list_connected_accounts`.
*   **Help & Support System:** Users can report bugs, errors, connection issues, feature requests, and general feedback from any AI agent chat via the Help button in the header. Tickets are categorized (bug/error/connection/feature/general) and tagged with the active agent. Users can view their ticket history and see admin replies. Admin panel has a "Support Tickets" tab to view all tickets, filter by status, reply to users, and update status/priority. DB table: `support_tickets` (extended with `category`, `agentType`, `adminReply` fields). APIs: `GET/POST /api/support-tickets` (user), `GET /api/admin/support-tickets`, `PATCH /api/admin/support-tickets/:id` (admin).
*   **Boss AI Notification System:** When AI agents send emails or complete tasks, Boss AI is notified. Notifications are stored in `boss_notifications` table. If user has Gmail configured, email notifications are sent to the user. Service: `server/bossNotificationService.ts`. API: `GET /api/boss/notifications`.
*   **PWA Support:** The application includes a web app manifest and service worker for PWA capabilities, offering an installable, app-like experience.
*   **Brand Identity Protection:** AI agents are programmed to never reveal underlying technologies, always attributing their capabilities to "proprietary RentAI 24 AI technology."

## External Dependencies
*   **OpenAI:** Utilized for core AI agent functionalities, specifically GPT-4o and `text-embedding-3-small`, integrated via Replit AI Integrations.
*   **PostgreSQL (Neon serverless):** Primary database solution for storing all application data, including user profiles, rentals, agent documents, and fine-tuning job details.
*   **Stripe:** Integrated for all payment processing, including subscriptions for AI worker rentals, one-time image credit purchases, and managing customer billing portals.
*   **Drizzle ORM:** Used for object-relational mapping with the PostgreSQL database.
*   **Framer Motion:** Employed for animations in the frontend.
*   **React Hook Form & Zod:** Used for form management and validation.
*   **TanStack Query:** Manages data fetching, caching, and synchronization in the frontend.
*   **Lucide React & React Icons/si:** Icon libraries for the user interface.
*   **Multer:** Middleware for handling `multipart/form-data`, primarily for file uploads (e.g., RAG documents).
*   **Google Fonts (Inter):** The chosen font for the application's typography.
*   **Gmail (OAuth) & Resend:** Used for email routing and sending, particularly for sales and customer support agents.
*   **Google Calendar:** Integrated for scheduling appointments by the Cal agent.
