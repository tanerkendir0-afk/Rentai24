# RentAI 24

## Overview
RentAI 24 is the world's first AI staffing agency website, offering businesses on-demand, pre-trained AI agents for roles like customer support, sales, and bookkeeping. The platform aims to streamline business operations by providing flexible AI talent, reducing traditional hiring complexities, and capitalizing on the increasing demand for AI in business. Key capabilities include managing a catalog of AI workers, user authentication, customer dashboards, admin panels, and a robust AI tooling system. The platform emphasizes token optimization, advanced security measures, and agent collaboration.

## System Architecture
The RentAI 24 platform utilizes a modern web stack with React, TypeScript, and Tailwind CSS for the frontend, bundled with Vite. The UI features a dark mode with navy and blue-to-violet gradients. The backend is built with Express.js, leveraging PostgreSQL (Neon serverless) and Drizzle ORM. OpenAI GPT-4o is integrated for core AI functionalities.

**Key Architectural Decisions and Features:**

*   **User Management:** Session-based authentication and authorization supporting `user`, `agent_manager`, and `admin` roles, including Google OAuth.
*   **AI Worker Management:** A catalog of 9 distinct AI agents with specific personas, managed through a customer dashboard and admin panel.
*   **AI Tooling System:** A generalized registry allowing AI agents to perform actions like sending emails or generating images.
*   **RAG & Fine-tuning:** Support for document uploading, chunking, embedding (pgvector storage), and OpenAI fine-tuning.
*   **Token Optimization:** Implements smart model routing, conversation history summarization, compressed system prompts, RAG threshold adjustments, and conditional tool filtering.
*   **Persistent Chat & Team Features:** Server-side chat persistence and a system for managing team members whose context is injected into agent prompts.
*   **Communication Integrations:** Per-user Gmail (Google OAuth) for email operations, social media account integration for auto-publishing, and WhatsApp Business integration for agent-driven messaging.
*   **Security & Monitoring:** Multi-layer AI guardrails (input/output), distillation protection, server hardening, obfuscated admin panel, and an admin security report. Server stability features: global error handlers (uncaughtException/unhandledRejection), graceful shutdown (SIGTERM/SIGINT), per-agent circuit breaker (3 failures ŌåÆ 5min cooldown in `server/services/circuitBreaker.ts`), per-user+agent rate limiting (20 msg/min), 30s request timeout on all AI API calls, agent heartbeat monitoring (`server/services/heartbeat.ts` ŌĆö checks all 9 agents every 5min for tool/DB availability without AI API calls, auto-triggers circuit breaker on failures), and health check endpoint (`GET /api/diagnostics/health` ŌĆö no auth required) reporting DB/OpenAI status, memory usage, uptime, active connections, circuit breaker states, and per-agent heartbeat status.
*   **Agent Behavior Enhancements:** Proactive agent behavior with a `web_search` tool, efficiency rules, and improved error handling.
*   **Manager Agent (Smart Router):** An intelligent routing layer that classifies user messages and directs them to the appropriate hired agent.
*   **Human Escalation System:** Detects frustrated user interactions, triggers escalations, and allows admins to join live chats.
*   **Agent Document Handling:** All agents can read, analyze, and correct user-uploaded documents, with specific enhancements for data analysis.
*   **Multi-AI Provider Support:** Allows selection of alternative AI providers (e.g., Anthropic Claude) per-agent from the admin panel, with A/B testing and spend comparison features.
*   **Finn Turkish Accounting Upgrade:** Specialized bookkeeping agent with professional Turkish accounting features including KDV invoicing (with DB persistence, PDF/Excel export), TCMB exchange rates, comprehensive financial management, and Excel report generation (Mizan, Bordro, Gelir Tablosu, KDV ├¢zet). Invoice data stored in `invoices` and `invoice_items` tables. Reports generated via `server/services/invoiceGenerator.ts`, `server/services/invoiceExcelGenerator.ts`, and `server/services/reportGenerator.ts`. Download endpoints: `GET /api/invoices/:id/pdf`, `GET /api/invoices/:id/excel`, `GET /api/reports/:reportId/download`. Finn's system prompt includes LaTeX math formatting instructions for calculations.
*   **KaTeX Math Rendering:** Chat messages support LaTeX math rendering via `remark-math` + `rehype-katex` plugins in `client/src/components/chat-message.tsx`. Inline math (`$...$`), block math (`$$...$$`), and LaTeX commands (`\frac{}`, `\times`, `\sum`, etc.) are rendered correctly.
*   **Finn Calculation Tools:** Dedicated calculation service (`server/services/calculationService.ts`) provides deterministic math for Finn instead of relying on LLM arithmetic. Six tools: `calculate_kdv` (KDV with tevkifat), `calculate_bordro` (full payroll with SGK/GV/AGI/BES), `calculate_amortisman` (normal/azalan depreciation tables), `calculate_kur_degerleme` (FX revaluation with journal entries), `calculate_stopaj` (withholding with KDV tevkifat and journal entries), `format_yevmiye` (journal entry formatter with balance check). All tools log actions via `storage.createAgentAction`. System prompt instructs Finn to always use tools instead of self-calculating.
*   **CRM Document Manager:** Allows users to upload and manage customer documents within the platform.
*   **KVKK/GDPR Compliance:** Full data privacy infrastructure including: `consent_logs` table tracking cookie/dataProcessing/kvkk consent with IP and user-agent; `cookieConsent` and `dataProcessingConsent` boolean fields on users; cookie consent banner (localStorage + API); KVKK checkbox on registration form; data export endpoint (`GET /api/user/data-export`); account deletion endpoint (`DELETE /api/user/account`) with cascading data removal; admin consent statistics panel showing consent rates and log history; all UI fully bilingual (EN/TR).
*   **User Behavior Analytics:** Internal analytics system (no external services) with KVKK consent enforcement. `page_views` and `user_events` tables track page visits and user actions. Frontend `AnalyticsProvider` (`client/src/lib/analytics.tsx`) auto-tracks page views on route changes using `navigator.sendBeacon` with proper JSON Blob content type, and exports `trackEvent()` for event logging. Only sends data when cookie consent is explicitly accepted. Backend routes (`POST /api/analytics/pageview`, `POST /api/analytics/event`) validate payloads with Zod schemas and check server-side `dataProcessingConsent`. Admin analytics dashboard (under Analytics > User Behavior tab) shows DAU, page views, top events, popular pages, agent usage breakdown, daily activity chart, and registration-to-rental conversion funnel with day/week/month period filtering. Key tracked events: `user_registered`, `chat_message_sent`, `agent_rented`.
*   **User Profile Enrichment & Onboarding:** Post-registration onboarding dialog (skippable) collecting optional fields: industry, company size, country, intended agents (multi-select), and referral source. All fields use validated enums. Data persists via `PATCH /api/auth/onboarding`. Enrichment fields are also editable in Settings profile section. Admin demographics panel shows distribution charts for industry, company size, country, referral source, and intended agents with onboarding completion metrics. Full EN/TR i18n support. Schema fields on `users` table: `industry`, `company_size`, `country`, `intended_agents` (text[]), `referral_source`, `onboarding_completed` (boolean).
*   **User Feedback System:** Multi-channel feedback collection with NPS surveys (0-10 scale, 30-day cadence, localStorage+backend scheduling), chat-end emoji ratings (4-level: ­¤ś×­¤śÉ­¤śŖ­¤ż® mapped to scores 2/5/8/10, per-session dedup), and general feedback form (categories: bug_report/feature_request/general). Backend uses discriminated union Zod validation per feedback type. `feedback` table stores all types with userId, type, score, comment, agentType, category. Storage methods: `createFeedback`, `getLastNpsByUser`, `getFeedbackList` (typed enum filter), `getFeedbackSummary` (NPS avg/trend, agent satisfaction, category distribution). Admin feedback dashboard under Analytics tab shows NPS average, total counts, chat satisfaction per-agent bars, category distribution badges, and filterable feedback list with user enrichment. Floating feedback button (bottom-left) and NPS widget (bottom-right, 3s delay) auto-appear for authenticated users. Full EN/TR i18n in `pages.json` under `feedback.*` keys. *   **Admin Agent Custom Instructions:** Enables admins to set global and per-agent custom instructions for runtime injection into agent system prompts.
*   **Brand Identity and Secrecy:** Agents attribute capabilities to "proprietary RentAI 24 AI technology" and refuse to reveal platform internals without a specific password.
*   **Internationalization (i18n):** Full bilingual support (English/Turkish) using react-i18next on frontend and a centralized message dictionary on backend. Default language is English. Users toggle TR/EN via a navbar button. Logged-in users' language preferences are saved to the database (`users.language` column) and restored on login. Locale files are in `client/src/locales/{en,tr}/{common,pages,agents}.json`. The `LanguageProvider` component (`client/src/lib/language.tsx`) wraps the app inside `AuthProvider` and syncs language from user data. Backend endpoint `PATCH /api/auth/language` persists the preference. Backend i18n: `server/i18n.ts` provides `resolveUserLang(req)` and a `langMiddleware()` that sets `req.lang` on all `/api` routes. `server/messages.ts` contains all bilingual error/status messages accessed via `msg(key, lang)`. All API error responses use `msg()` for bilingual support. Email templates (`server/emailTemplates.ts`) and guardrails (`server/guardrails.ts`) also support bilingual output.

*   **Stability & Diagnostics:** Process-level crash handlers (`uncaughtException`, `unhandledRejection`), startup memory logging, diagnostics health endpoint (`GET /api/diagnostics/health`) returning uptime/memory/connections, and a comprehensive test script (`scripts/diagnostics-test.ts`) covering crash resilience, port verification, and memory leak detection.
